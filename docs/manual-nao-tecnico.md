# Manual Não Técnico — Sistema Laboral (RPVP)

Este manual foi preparado para pessoas de fora da área de tecnologia, como diretoria, gestores, instrutores e colaboradores. O objetivo é explicar de forma simples o que o sistema faz, como ele é usado no dia a dia e quais cuidados são importantes para o uso correto.

---

## 1. O que é o sistema

O **Sistema Laboral (RPVP)** é um sistema usado para **registrar presença com reconhecimento facial** e acompanhar sessões realizadas pela empresa.

Na prática, ele concentra em um único lugar:

- o acesso dos usuários ao sistema;
- o registro de presença com câmera;
- o acompanhamento das sessões em andamento;
- a consulta de registros e auditorias;
- a administração de usuários, tipos de sessão e configurações.

## 2. Para que ele serve

O sistema ajuda a empresa a organizar melhor o processo de presença. Em vez de depender apenas de controles manuais, ele permite:

- registrar a presença de uma pessoa por reconhecimento facial;
- registrar entrada e, quando a sessão exigir, também a saída;
- acompanhar quem participou de cada sessão;
- consultar históricos com filtros;
- gerar relatórios em PDF e Excel;
- manter usuários e fotos atualizados para reconhecimento.

## 3. Quem usa o sistema

O sistema trabalha com quatro perfis principais:

| Perfil | Papel no sistema |
|---|---|
| **Admin** | Cuida da administração geral do sistema. |
| **Gestor** | Consulta informações operacionais e auditorias. |
| **Instrutor** | Cria e conduz sessões, além de acompanhar os registros das próprias sessões. |
| **Colaborador** | Usa o sistema para registrar presença. |

Cada perfil enxerga e utiliza apenas o que faz sentido para sua função.

---

## 4. Principais áreas do sistema

### 4.1 Login

O acesso ao sistema é feito com usuário e senha.

No primeiro acesso, o sistema pode exigir a troca da senha antes de liberar o restante do uso. Isso aumenta a segurança e evita que a senha inicial continue sendo usada.

### 4.2 Registro de presença por reconhecimento facial

Essa é a função principal do sistema.

De forma simples, o processo acontece assim:

1. a pessoa entra na tela de presença;
2. seleciona a sessão correta;
3. posiciona o rosto na câmera;
4. realiza a piscada solicitada;
5. o sistema confirma o registro.

Dependendo da configuração da sessão, o sistema pode registrar:

- apenas a entrada;
- entrada e saída.

### 4.3 Sessões

O sistema permite organizar as sessões realizadas pela empresa.

Nessa parte, é possível:

- criar uma nova sessão;
- definir o tipo da sessão;
- informar local e observações;
- iniciar a sessão;
- encerrar a sessão;
- acompanhar em tempo real quem já registrou presença.

### 4.4 Dashboard

O sistema possui um painel com visão rápida da operação. Esse painel ajuda a acompanhar o dia, ver o andamento das sessões e consultar informações de forma mais rápida.

### 4.5 Auditoria de registros

Existe uma área própria para consulta dos registros já realizados.

Nela, é possível:

- pesquisar por filtros;
- consultar histórico de presenças;
- ver status de registros;
- visualizar a imagem ligada ao registro, quando disponível;
- exportar informações em **PDF** ou **Excel**.

### 4.6 Usuários

O sistema também possui uma área administrativa de usuários.

Essa área permite:

- cadastrar novos usuários;
- editar dados;
- definir o perfil de acesso;
- ativar ou desativar usuários;
- marcar troca obrigatória de senha no primeiro acesso;
- vincular gestor e setor quando necessário.

### 4.7 Coleção facial

Cada usuário pode ter fotos cadastradas para reconhecimento facial.

Essas fotos formam a **coleção facial** usada pelo sistema para identificar a pessoa.

Na prática, é possível:

- enviar fotos;
- consultar as imagens já cadastradas;
- adicionar novas imagens;
- remover imagens específicas;
- limpar toda a coleção facial do usuário.

### 4.8 Configurações e tipos de sessão

O sistema possui uma área administrativa para ajustes gerais.

Nela, a empresa pode:

- ajustar regras de presença;
- configurar parâmetros do reconhecimento facial;
- definir limites de operação;
- cadastrar, editar e remover tipos de sessão.

Esses ajustes ajudam a adaptar o sistema à rotina da empresa sem necessidade de alterar o programa manualmente.

## 5. Como o sistema é usado no dia a dia

### 5.1 Para o colaborador

O uso normal acontece assim:

1. fazer login;
2. se for o primeiro acesso, trocar a senha;
3. escolher a sessão correta;
4. olhar para a câmera;
5. piscar quando solicitado;
6. aguardar a confirmação do registro.

### 5.2 Para o instrutor

O instrutor pode:

- criar a própria sessão;
- iniciar a sessão no momento correto;
- acompanhar quem já registrou presença;
- encerrar a sessão ao final;
- exportar os dados da sessão quando necessário.

### 5.3 Para o gestor

O gestor usa o sistema principalmente para:

- acompanhar o andamento das sessões;
- consultar registros e auditorias;
- verificar informações operacionais.

### 5.4 Para o administrador

O administrador atua na manutenção geral do sistema. Isso inclui:

- cadastro de usuários;
- definição de perfis;
- manutenção da coleção facial;
- configuração do sistema;
- gestão de tipos de sessão;
- acompanhamento de auditorias e relatórios.

- mais rapidez no registro de presença;
- menos dependência de controles manuais;

## 8. Pontos de atenção

Para o sistema funcionar bem, alguns cuidados são importantes.

### 8.1 O que é necessário para funcionar

- usuário com acesso válido;
- câmera disponível no dispositivo;
- sessão correta selecionada antes do registro;
- fotos adequadas na coleção facial do usuário;
- ambiente com infraestrutura funcionando normalmente.

### 8.2 Cuidados no uso

- o usuário precisa entrar na sessão certa antes de registrar presença;
- se a sessão trabalhar com saída, pode existir um tempo mínimo entre entrada e saída;
- sessões encerradas não aceitam novos registros;
- fotos ruins ou desatualizadas podem prejudicar o reconhecimento;
- o primeiro acesso pode exigir troca obrigatória de senha.

### 8.3 O que fazer quando houver falha no reconhecimento

Se o sistema não reconhecer o rosto, o ideal é verificar:

- se a sessão correta foi selecionada;
- se a câmera está funcionando bem;
- se o rosto está bem posicionado;
- se as fotos do usuário estão atualizadas na coleção facial.

Se o problema continuar, o caso deve ser encaminhado para quem administra o sistema.

## 9. Perguntas frequentes

### 1. Qual é a principal função do sistema?
Registrar presença com reconhecimento facial e organizar as sessões da empresa.

### 2. O sistema registra só entrada?
Não. Em algumas sessões ele também registra a saída.

### 3. O usuário precisa escolher a sessão antes do registro?
Sim. A presença fica vinculada à sessão selecionada.

### 4. O que acontece no primeiro acesso?
O sistema pode exigir que a senha seja trocada antes de continuar o uso.

### 5. É possível acompanhar uma sessão em tempo real?
Sim. O sistema possui tela de acompanhamento da sessão.

### 6. É possível consultar registros antigos?
Sim. Existe uma área de auditoria com filtros de pesquisa.

### 7. O sistema permite exportar relatórios?
Sim. Os dados podem ser exportados em PDF e Excel.

### 8. O que é coleção facial?
É o conjunto de fotos usado pelo sistema para reconhecer cada usuário.

### 9. Quem pode alterar configurações do sistema?
Normalmente essa função fica com o perfil administrativo.

### 10. O sistema também ajuda na gestão?
Sim. Ele oferece acompanhamento, auditoria, relatórios e organização das sessões.

## Encerramento

Em resumo, o **Sistema Laboral (RPVP)** é um sistema de **controle de presença com reconhecimento facial** que também apoia a organização das sessões, a consulta de históricos e a administração da operação.

Este manual foi reescrito em linguagem simples para facilitar a leitura por pessoas não técnicas, mantendo alinhamento com o funcionamento atual do sistema.
